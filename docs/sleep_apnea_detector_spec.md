# Спецификация детектора подозрения на апноэ сна

## Цель сигнала

Выявить по данным Apple Health (в первую очередь Apple Watch) **вероятностный** ночной риск, связанный с эпизодами снижения частоты дыхания во сне при поддержке со стороны пульса и/или HRV. Сигнал предназначен для навигации пользователя к врачу, а не для постановки диагноза.

## Ограничения источника данных Apple Health

- Нет прямого измерения дыхательных событий уровня полисомнографии.
- Частота дыхания (RR), пульс и HRV приходят дискретными выборками с переменным шагом.
- Сегменты сна задаются записями сна; вне сна дыхательные точки в алгоритме **не используются**.
- Качество сигнала зависит от ношения часов ночью и синхронизации.

## Что такое подозрительная точка

**Подозрительная точка дыхания:** запись частоты дыхания (respiratory rate) внутри сегмента сна, для которой **RR &lt; 10** (в тех же единицах, что в HealthKit).

## Подтверждённая подозрительная точка

Подозрительная точка дыхания **и** выполняется хотя бы одно из условий:

- ближайший по времени HR (в окне **±120 с**) **не ниже** ночного baseline HR **минимум на 12 уд/мин**;
- ближайший по времени HRV (в окне **±120 с**) **не выше** ночного baseline HRV **минимум на 20%** (то есть `hrv ≤ baseline_hrv × 0.8`).

Точка помечается флагами поддержки `supported_by_hr` и/или `supported_by_hrv`.

## Что такое подозрительный эпизод

Группа **подтверждённых** подозрительных точек, упорядоченных по времени.

## Как точки объединяются в эпизоды

- Учитываются только подтверждённые точки.
- Соседние по времени точки относятся к одному эпизоду, если разрыв между их временными метками **не более 5 минут** (300 с).
- Любой разрыв **более 5 минут** **запрещает** склейку: текущий эпизод заканчивается, следующий начинается с новой точки.

## Валидность эпизода

Эпизод считается валидным, если:

- в нём **не менее 2** подтверждённых точек;
- длительность по времени от первой до последней точки **не менее 2 минут** (120 с).

## Уровень поддержки эпизода (`support_level`)

- **strong** — в эпизоде есть и поддержка по HR, и по HRV (по совокупности точек).
- **medium** — есть поддержка **только** по одному из каналов (только HR или только HRV).
- **weak** — нет ни одной поддержки (в реализации не встречается для подтверждённых точек).

В **итоговый скор риска** входят только эпизоды с уровнем **medium** или **strong**.

## Базовые линии (ночь, внутри сна)

- `baseline_rr` — медиана всех ночных RR-точек с **RR ≥ 10**.
- `baseline_hr` — медиана всех ночних HR-точек внутри сна.
- `baseline_hrv` — медиана всех ночних HRV-точек внутри сна.

## Недостаточно данных (`insufficient data`)

Возвращается без построения эпизодов/скора, если выполняется любое из условий:

- нет сегментов сна в выборке;
- суммарная длительность сна по сегментам **&lt; 4 часов**;
- ночных RR-точек **меньше 20**;
- нет RR ≥ 10 для расчёта `baseline_rr` (медиана по пустому множеству).

## Преобразование эпизодов в ночной риск (score)

Параметры только по эпизодам **medium/strong**:

- `episode_density = valid_episode_count / sleep_hours`
- `density_component = min(1.0, episode_density / 5.0)`
- `support_component = min(1.0, strong_episode_count / max(1, valid_episode_count))`
- `median_resp_drop` — медиана по эпизодам величины `(baseline_rr − median_rr_эпизода)`, где `median_rr_эпизода` — медиана RR в эпизоде
- `depth_component = min(1.0, median_resp_drop / 6.0)` (с отсечением снизу нуля при необходимости)

**Итог:**

`score = 0.5 × density_component + 0.3 × support_component + 0.2 × depth_component`

## Confidence

- `rr_coverage_component = min(1.0, respiratory_points / 40.0)`
- `cross_signal_component = min(1.0, (heart_points + hrv_points) / max(1, respiratory_points))`
- `sleep_hours_component = min(1.0, sleep_hours / 7.0)`

`confidence = 0.4 × rr_coverage + 0.35 × cross_signal + 0.25 × sleep_hours`

## Поля в БД (`sleep_apnea_events`)

Дополнительно к уже существующим полям события сохраняют:

- `point_count` — число подтверждённых точек в эпизоде;
- `support_level` — `strong` / `medium` / `weak`;
- `confidence` — достоверность расчёта на момент детекции (общая по прогону);
- `baseline_rr`, `baseline_hr`, `baseline_hrv` — ночные baseline;
- `sleep_hours_context` — суммарные часы сна в контексте расчёта.

## Чем алгоритм не является

- **Не** клинический диагноз и **не** замена очной оценке сомнолога.
- **Не** замена полисомнографии.
- **Не** расчёт AHI и не эквивалент клиническому индексу апноэ.

## Полный алгоритм (вход и шаги)

**Вход:** строки respiratory rate, heart rate, HRV, список сегментов сна `(start, end)`.

1. Отфильтровать RR, HR, HRV — оставить только точки **внутри** сегментов сна.
2. Если суммарный сон **&lt; 4 ч** или RR-точек **&lt; 20** — **insufficient data**.
3. Посчитать `baseline_rr` (медиана RR при RR ≥ 10), `baseline_hr`, `baseline_hrv` по ночным точкам.
4. Найти подозрительные точки (RR &lt; 10), для каждой — ближайший HR/HRV в ±120 с; пометить подтверждение и флаги поддержки.
5. Построить эпизоды по подтверждённым точкам с максимальным разрывом **5 минут** между соседними RR-точками.
6. Отфильтровать валидные эпизоды (≥2 точек, длительность ≥2 мин).
7. Назначить `support_level`; для скора оставить только **medium** и **strong**.
8. Вычислить `episode_density`, компоненты и **score**, затем **confidence** по формулам выше.
